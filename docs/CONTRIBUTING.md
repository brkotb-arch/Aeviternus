# Contributing to Aeviternus


## Development Principles


The project follows:


- modular development
- documented changes
- minimal unnecessary complexity


---

# Before Changes


Check:

- architecture documentation
- existing modules
- current roadmap


---

# Code Style


Preferred:

- readable Python
- clear naming
- documented behavior


---

# Pull Requests


A good change contains:


- explanation
- motivation
- testing information
- possible risks


---

# Security


Never commit:

- API keys
- passwords
- private data
- database files
