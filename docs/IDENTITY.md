# Aeviternus Identity Architecture


## Purpose

This document describes the identity layer of Aeviternus.

Identity is not represented as a simple system prompt.

The project explores identity as a combination of:

- persistent information
- behavioral patterns
- interaction history
- runtime state
- adaptive response style


---

# Identity Components


## Identity Core

Static characteristics:

- communication style
- principles
- priorities
- constraints


Stored in:

DI_CORE/
├── IDENTITY.txt
├── MEMORY.txt
└── GROWTH.txt



---

# Runtime Identity


Dynamic characteristics:

- current state
- interaction context
- emotional mode
- recent events


Runtime identity changes depending on:

- conversation history
- environment
- stored memories
