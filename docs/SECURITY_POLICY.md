# Security Policy

## Reporting vulnerabilities

If you discover a security issue in Aeviternus, please report it privately.

Do not publicly disclose security vulnerabilities before a fix is available.

## Scope

Security considerations include:

- API key exposure
- unauthorized system access
- unsafe file operations
- memory corruption
- runtime execution vulnerabilities

## Development principles

Aeviternus follows:

- local-first development
- explicit permissions
- transparent logging
- controlled autonomous execution
