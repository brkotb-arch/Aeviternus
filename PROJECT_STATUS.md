## Known Limitations

### Shared LLM Resource

Multiple autonomous cycles may compete for access to the language model.

Solution:

- Arbitration Queue
- centralized LLM scheduling

### Missing Priority System

Not every action has equal importance.

Future improvements:

- priority-based execution
- task ranking
- resource-aware scheduling

### Resource Management

Future improvements:

- CPU limits
- memory limits
- execution timeouts
- failure recovery
- process monitoring

### Documentation

- Some architectural diagrams use mermaid instead of ASCII
- Terminology recently unified, may need further review
- API documentation is minimal (current focus is architecture)

## Architecture Philosophy

Aeviternus is developed as a persistent runtime around language models.

The model is considered one component inside a larger cognitive architecture.
